Several issues found during testing across all three roles:
Issue 1 — Time picker not handling AM/PM correctly: When entering a start time of 12:00 and an end time of 2:00, the system incorrectly flags the end time as invalid, treating 12 as numerically greater than 2. The time input needs to be replaced with a proper time picker that handles hours, minutes, and AM/PM correctly so that 12:00 PM to 2:00 PM is recognized as a valid range.
Issue 2 — Active listings not showing on the browse page: When logged in as a student account (sdalal@depaul.edu or similar), navigating to the main browse page shows no active listings at all. The search bar and category filters appear to work, but the active listings themselves are not rendering in the feed. This is a data loading issue on the browse screen, not a filtering issue.
Issue 3 — Remove listing not working for org accounts: When logged in as the Robotics Club, tapping the remove button on a pending or active listing does nothing. The button appears tappable but the action does not execute and no feedback is shown to the user. The DELETE /api/listings/:id endpoint or the handler calling it is likely not firing correctly.
Issue 4 — Approve and reject not working on the admin approval queue: When logged in as admin, the pending listings queue loads correctly. However, tapping Approve does nothing at all. Tapping Reject opens the reason input modal correctly, but after typing a reason and confirming, the rejection does not go through. Both actions are silently failing — no success message, no error message, no status change. The PATCH /api/listings/:id/status endpoint is likely returning an error that is not being surfaced to the UI.
Issue 5 — Pending count showing zero in activity dashboard: The activity dashboard shows zero pending listings even though several are actually pending. The stats endpoint or the component rendering the pending count is not reading the correct value.
Issue 6 — Org account approval and rejection not working from admin: When a new organization account (CSS) was created, navigating to the org verification screen and attempting to approve or deny the account does not work. The action does not complete and the org's status does not change. The PATCH /api/users/orgs/:orgId/verify endpoint or the handler calling it is not executing correctly.
